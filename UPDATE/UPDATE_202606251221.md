## 1. Model Training Pipeline (`gemini-training.py`)
This script handles the data preprocessing, feature engineering, and the training of the CNN-LSTM Base Model and TCN Gatekeeper. 

**Recent Core Upgrades:**
* **Same Bar Conflict Resolution:** Implemented a strict conservative logic for data labeling. If market volatility causes both the Take Profit (+3 ATR) and Stop Loss (-2 ATR) barriers to be breached within the exact same 5-minute candle, the labeler now automatically defaults to the Stop Loss. This prevents optimistic Look-Ahead Bias and forces the neural network to train on the worst-case scenario.
* **Performance Telemetry:** Added execution timing metrics. The script now records the **Start Time**, **End Time**, and **Total Duration** of the training cycle. This allows for accurate hardware benchmarking and efficiency tracking when retraining models over larger datasets.
* **ONNX Export:** Models are successfully compiled into highly optimized `.onnx` formats for ultra-fast, CPU-bound live inference.

## 2. Vectorized Simulation Engine (`backtest_simulator.py`)
This script serves as the rapid, Pandas-based historical testing environment to validate the mathematical edge of the AI before live deployment.

**Recent Core Upgrades:**
* **Strategy Selection Logic:** Integrated a master toggle allowing the system to seamlessly switch between a `FIXED` lot allocation and an aggressive `DYNAMIC` compounding algorithm (e.g., $100 equity increments = +0.01 lots).
* **Execution Lock Mechanism:** Added an `in_trade` state tracker. If a trade is currently active, the simulator will ignore subsequent AI signals until the Take Profit or Stop Loss is struck, perfectly mirroring live margin-protection constraints.
* **MT5 Signal Injector (Bridge):** Built an automated CSV exporter (`AI_Signals_XAUUSD.csv`) that applies a strict $t+1$ temporal shift. This allows the discrete Python signals to be imported into the MetaTrader 5 Strategy Tester for continuous, tick-by-tick visual validation, proving the absence of data leakage.

This is the production-grade deployment script. It acts as a continuous, event-driven state machine connecting the trained ONNX models directly to the MetaTrader 5 terminal via the official Python API. It handles data synchronization, CPU inference, and real-time macro-execution.

**Architectural Deep-Dive:**

### 3.1 The Asynchronous Heartbeat (Micro-Polling)
Instead of utilizing blind "deep sleep" functions, the system maintains a highly efficient `0.5s` active micro-polling loop. 
* **Modulo Synchronization:** The script calculates `current_time % 300` against the system clock. It only triggers the heavy Machine Learning pipeline when a 5-minute block has perfectly elapsed.
* **Data Buffer:** A strict 2-second synchronization buffer is enforced before downloading data, guaranteeing the broker has finalized the OHLC and volume calculations for the closing candle.

### 3.2 Defensive Shields (State Management)
Before the AI evaluates the market, the script verifies environmental integrity:
* **Stale Candle Protection:** The system records the timestamp of every processed candle. If the broker pushes a candle with an identical timestamp (indicating a weekend, holiday, or frozen market), the script safely hibernates, preventing API request spam and false executions.
* **Execution Lock Check:** The system queries live account positions. If the strategy parameter `ALLOW_MULTIPLE_TRADES` is `False`, the script acts as a hard gatekeeper, ignoring new AI signals while a trade is active to protect account margin.

### 3.3 The Dual-Inference Engine
Once a fresh candle is verified, the script runs an identical Pandas feature-engineering pipeline to the training environment and applies the `scaler.pkl`. It then triggers a logical gate:
* **Model A (CNN-LSTM):** Evaluates the 60-candle sequence to predict direction (Buy/Sell/Hold).
* **Model B (TCN Gatekeeper):** Evaluates the sequence to generate a confidence probability. 
* **Routing:** A trade is only routed if the CNN-LSTM detects a signal AND the TCN confidence score exceeds a strict `52%` threshold.

### 3.4 Telemetry & Macro-Execution
* **Dynamic TBM Injection:** The script extracts the live Average True Range (ATR) from the final candle, dynamically calculating exact price coordinates for the +3 ATR Take Profit and -2 ATR Stop Loss before routing the order.
* **Real-Time Accounting:** The 0.5s polling loop continuously compares currently active trade tickets against memory (`active_trade_tickets`). If a ticket vanishes (hits TP/SL), it instantly extracts the broker receipt.
* **Telegram Notification System:** Pushes formatted Markdown alerts to a mobile device for:
  1. Executed Trades (Asset, Direction, Volume, TP/SL, Running Equity).
  2. Closed Trades (Net PnL, New Account Balance).

### 3.5 Global Crash Handler
The entire execution architecture is wrapped in a comprehensive `try/except` block with persistent file logging (`trading_bot.log`). Any fatal Python exceptions will generate a localized traceback log, push a critical SOS Telegram alert to the administrator, and safely sever the MT5 connection before terminating.
