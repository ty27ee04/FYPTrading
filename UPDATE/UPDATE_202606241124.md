## V3.0 Production-Ready Deployment Upgrades

### 1. Market Reality Friction (Spread Mechanics)
* **The Issue:** The previous simulation environment assumed frictionless fills, executing trades at exactly the open, high, or low price. 
* **The Solution:** Upgraded the backtest engine to deduct a static $0.20 spread penalty from the raw price delta on every closed trade (`p_diff_net = p_diff_raw - spread_penalty`). This ensures the final equity curve mathematically reflects real-world broker transaction costs and proves the system maintains positive expectancy despite friction.

### 2. Model Portability (ONNX Export Pipeline)
* **The Issue:** Native PyTorch weights (`.pth`) require heavy library dependencies (CUDA, PyTorch) to run inference, making live server deployment or Expert Advisor (EA) integration inefficient.
* **The Solution:** Engineered an automated ONNX (Open Neural Network Exchange) export block at the end of the training pipeline. Both the CNN-LSTM and TCN models are compiled into `best_model_a_live.onnx` and `best_model_b_live.onnx`, enabling high-speed, lightweight C++ or Python API inference on live MetaTrader5 environments.

### 3. Backtest Alignment with Triple Barrier Mechanics
* **The Issue:** A structural gap existed between the training objective and the execution objective. The neural network was trained to predict if price would hit a Take Profit (TP) or Stop Loss (SL) barrier, but the backtest engine was ignoring these barriers, buying on a signal and holding until the signal inverted.
* **The Solution:** Completely rewrote the `run_detailed_backtest` engine. When a signal is generated, the simulation now actively sets a `+3 ATR` Profit Target and `-2 ATR` Stop Loss line on the chart. The trade remains open and floating until one of these physical price barriers is crossed, or the 24-bar Time Stop (Max Horizon) is reached. This aligns the execution logic perfectly with the mathematical training objective.

### 4. Forensic Trade Logging (CSV Enhancement)
* **The Issue:** The generated `trade_log.csv` lacked the granular financial tracking required for institutional analysis and FYP thesis defense.
* **The Solution:** Expanded the dictionary payload appended during trade exits. The CSV now forensicly tracks: `Trade_ID`, `Entry/Exit Times`, `Direction`, exact `Entry/Exit Prices`, the dynamic `Lot_Size`, the physical `Spread_Charge_USD` deducted, the `Net_PnL_USD`, the exact `Exit_Reason` (e.g., "Take Profit" vs "Time Stop"), and the updated `Running_Equity_USD` balance.
